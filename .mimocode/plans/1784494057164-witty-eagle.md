# Como Compilar o Projeto Veloren

## Resumo

Veloren é um jogo voxel RPG open-source escrito em **Rust** (edition 2024), toolchain **nightly-2026-06-13**. Usa **Cargo workspace** com 27 crates.

## Pré-requisitos

1. **Rust nightly** — `rustup` seleciona automaticamente o toolchain correto via `rust-toolchain`
2. **Git + Git LFS** — `git lfs install && git lfs pull` para baixar assets grandes (`.vox`, `.jpg`, `.wav`)
3. **No Linux (para o client gráfico):** dependências de sistema (X11, Wayland, Vulkan, ALSA, etc.) — ver abaixo

## Comandos de Compilação

### Build de Desenvolvimento (padrão)
```bash
cargo build
```

### Rodar o Client (voxygen)
```bash
cargo run --bin veloren-voxygen
# ou via alias:
cargo test-voxygen
```

### Rodar o Servidor
```bash
cargo run --bin veloren-server-cli
# ou via alias:
cargo server
```

### Build de Release (otimizado)
```bash
# Linux x86_64
export VELOREN_USERDATA_STRATEGY=executable
cargo build --release --no-default-features --features default-publish

# Windows
cargo build --target=x86_64-pc-windows-gnu --release --no-default-features --features "default-publish"

# macOS x86_64
cargo build --profile release-thinlto --no-default-features --features default-publish --target x86_64-apple-darwin

# macOS aarch64 (Apple Silicon)
cargo build --profile release --no-default-features --features default-publish --target aarch64-apple-darwin
```

### Testes
```bash
export VELOREN_ASSETS="$(pwd)/assets"
cargo test
```

### Lint / Formatação
```bash
cargo clippy --all-targets --locked --features="bin_cmd_doc_gen,bin_compression,bin_csv,bin_graphviz,bin_bot,bin_asset_migrate,bin,stat,cli" -- -D warnings
cargo fmt --all -- --check
```

## Dependências Linux (Ubuntu/Debian)

```bash
sudo apt install pkg-config cmake python3 libasound2-dev libxkbcommon-dev libudev-dev \
  libxcb1-dev libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev \
  libx11-dev libxi-dev libxcursor-dev libxrandr-dev libwayland-dev \
  libvulkan-dev libfontconfig1-dev
```

Também é necessário o linker **mold** (configurado em `.cargo/config.toml`), ou remova a flag `-fuse-ld=mold` desse arquivo.

## Variáveis de Ambiente Úteis

| Variável | Propósito |
|---|---|
| `VELOREN_ASSETS` | Caminho para a pasta de assets |
| `VELOREN_USERDATA_STRATEGY` | `executable` (ao lado do binário) ou `system` (caminhos do OS) |
| `SHADERC_LIB_DIR` | Caminho para biblioteca shaderc pré-compilada |

## Nix (alternativa)

```bash
nix build                    # Build do client (release)
nix build .#veloren-server-cli
nix develop                  # Shell de desenvolvimento
```

## Perfis de Build Disponíveis

| Perfil | Uso |
|---|---|
| `dev` | Desenvolvimento padrão (opt-level=2) |
| `debuginfo` | Desenvolvimento com símbolos de debug |
| `release` | Produção (LTO, opt-level=3) |
| `release-thinlto` | Release com Thin LTO |

## Arquivos de Configuração Relevantes

- `Cargo.toml` — workspace root com 27 crates
- `rust-toolchain` — fixa o toolchain nightly
- `.cargo/config.toml` — aliases e configurações de linker
- `.gitlab-ci.yml` — pipeline de CI/CD
- `flake.nix` — build via Nix
